# Proposal: Combined PR + PLIP Contributor Levels

**Date**: 2025-11-03
**Author**: Plone Contributor Statistics Analysis
**Status**: Draft Proposal

## Executive Summary

This proposal recommends a **unified contributor level system** that recognizes both:
- **Strategic contributions** (PLIPs - feature planning and architecture)
- **Implementation contributions** (PRs - code execution and delivery)

Rather than separate level systems, we propose a **3-tier framework** with **4 recognition tracks** that acknowledge different contribution patterns.

## Recommended: 3-Tier System with Recognition Tracks

| **Platinum** | Core Leaders | See matrix below | 🏆 |
| **Gold** | Active Contributors | See matrix below | ⭐ |

Within each tier, contributors can achieve recognition via **different tracks**:

- **🎯 Strategic Track**: PLIP-focused

- **🌟 Balanced Track**: Both PRs and PLIPs
- **🚀 Rising Star Track**: New contributors with high velocity

## Rationale for Combined System

### 1. Recognizes Different Contribution Styles

**Problem**: Separate systems create false hierarchy
- PLIPs seem "more important" than PRs
- Implementation work undervalued vs. strategic planning

**Solution**: Equal recognition for different contribution types
- Strategic architects get same tier as implementation champions
- Multiple paths to each tier

### 2. Encourages Balanced Growth

**Problem**: Contributors may specialize too narrowly

**Solution**: Balanced track provides higher recognition for versatility
- Motivates PLIP authors to implement
- Motivates PR contributors to propose features
- Creates well-rounded leaders

### 3. Data-Driven Thresholds

**Analysis of 2024 + Historical Data**:

| Metric | Platinum | Gold | Silver |
|--------|----------|------|--------|
| **PLIPs (all-time)** | 10+ (8 people) | 3-9 (22 people) | 1-2 (48 people) |
| **PRs (annual)** | 200+ (4 people) | 50-199 (8 people) | 10-49 (21 people) |
| **PRs (3-year)** | 500+ (5 people) | 100-499 (varies) | <100 |

**Natural breaks exist at**:
- PLIPs: 10, 3, 1
- Annual PRs: 200, 50, 10
- 3-year PRs: 500, 100

### 4. Motivational Progression

Clear paths for advancement:
- Silver → Gold: Submit 2+ more PLIPs OR increase to 50+ PRs/year
- Gold → Platinum: Reach 10 PLIPs OR sustain 200+ PRs/year OR balance both
- Multiple achievable routes, not single narrow path

## Time-Based Considerations

### Annual vs. All-Time Metrics

**PLIPs**: All-time count
- Rationale: Strategic vision has lasting impact
- 10-year-old PLIP still shows architectural thinking

**PRs**: Rolling 3-year window (or annual for active contributors)
- Rationale: Reflects current engagement level
- Prevents inactive contributors from holding Platinum
- Option: Use annual OR 3-year average, whichever qualifies

### Tier Review Frequency

**Annual Review**:
- Re-evaluate all tiers based on past 3 years activity
- Contributors can move up or down
- Platinum requires sustained activity (200+ PRs/yr OR active PLIPs)

**Grace Period**:
- Contributors get 1-year grace after dropping below threshold
- Prevents demotion due to sabbatical/personal leave
- After grace period, move to appropriate tier

## Frequently Asked Questions

### Q: I contribute through other means (documentation, community support). How am I recognized?

**A**: This system focuses on code contributions (PRs) and feature planning (PLIPs). We recommend:
- Adding a **Community** track in future iterations
- Expanding metrics to include documentation commits, forum participation, etc.
- For now, documentation PRs count toward Implementation track

### Q: What if I'm Platinum in PLIPs but Gold in PRs?

**A**: You're **Platinum** overall, with 🎯 Strategic track badge. Your tier is the highest you qualify for across any track.

### Q: Can I display multiple track badges?

**A**: Yes! If you qualify for multiple tracks at your tier:
- Platinum All-Rounder qualifies for all three (🎯⚙️🌟)
- Display all earned badges on profile
- Primary badge is highest-scoring track

### Q: How are 3-year metrics calculated for new contributors?

**A**:
- **Option 1**: Use annualized rate (e.g., 150 PRs in 1 year → 450 over 3 years projected)
- **Option 2**: Rising Star track exists for exceptional newcomers
- After 3 years, use actual 3-year data

### Q: What happens if I take a break from Plone?

**A**:
- **1-year grace period**: Tier maintained even if below threshold
- **After grace period**: Move to appropriate tier based on recent activity
- **Return**: Can quickly re-advance based on new contributions
- **Emeritus Status**: Option for long-time Platinum contributors who step back

### Q: How do organization contributions vs. individual contributions factor in?

**A**: Levels are individual-based only. Organization stats are tracked separately but don't affect individual tier.

## Alternative Considerations

### Option: Weighted Scoring System

Instead of multiple tracks, use single weighted score:

**Score = (Annual PRs × 1) + (All-time PLIPs × 50)**

- Platinum: 500+ points
- Gold: 150-499 points
- Silver: < 150 points

**Pros**: Simpler, single number
**Cons**: Loses nuance of contribution types, harder to explain

**Recommendation**: Multi-track system better recognizes diversity

### Option: 4-Tier System

Add **Diamond** tier for exceptional contributors:
- Diamond: 20+ PLIPs OR 1000+ PRs (3yr) OR 10+ PLIPs + 500+ PRs (3yr)
- Current qualifiers: tisto (61 PLIPs), sneridagh (30 PLIPs, 1046 PRs)

**Pros**: Recognizes truly exceptional contributors
**Cons**: Too granular, only 2-3 people currently qualify

**Recommendation**: Stick with 3 tiers, use track differentiation instead

## Recommendation

**We strongly recommend adopting the 3-Tier, Multi-Track system** because:

1. ✅ **Fair and Inclusive**: Values all contribution types equally
2. ✅ **Data-Driven**: Thresholds based on actual distribution patterns
3. ✅ **Motivational**: Multiple clear paths to advancement
4. ✅ **Flexible**: Accommodates different contributor styles
5. ✅ **Scalable**: Works as community grows
6. ✅ **Actionable**: Enables targeted programs for each tier/track

This system recognizes that **great communities need both visionaries and builders**, and celebrates both equally.

## Appendix: Tier Assignment Algorithm

```python
def assign_tier_and_track(contributor):
    """
    Assign tier and track based on PR and PLIP counts
    """
    annual_prs = contributor.prs_2024
    three_year_prs = contributor.prs_2022_to_2024
    total_plips = contributor.plips_alltime

    tier = None
    tracks = []

    # Platinum checks
    if total_plips >= 10:
        tier = "Platinum"
        tracks.append("Strategic")

    if three_year_prs >= 500 or annual_prs >= 200:
        tier = "Platinum"
        tracks.append("Implementation")

    if total_plips >= 5 and three_year_prs >= 300:
        tier = "Platinum"
        tracks.append("Balanced")

    # Gold checks (if not Platinum)
    if tier is None:
        if 3 <= total_plips <= 9:
            tier = "Gold"
            tracks.append("Strategic")

        if 100 <= three_year_prs < 500 or 50 <= annual_prs < 200:
            tier = "Gold"
            tracks.append("Implementation")

        if total_plips >= 2 and annual_prs >= 50:
            tier = "Gold"
            tracks.append("Balanced")

        if contributor.first_year and (annual_prs >= 50 or total_plips >= 2):
            tier = "Gold"
            tracks.append("Rising Star")

    # Silver (default)
    if tier is None:
        tier = "Silver"
        if 1 <= total_plips <= 2:
            tracks.append("Strategic")
        elif annual_prs >= 10:
            tracks.append("Implementation")
        else:
            tracks.append("Newcomer")

    return tier, tracks
```
